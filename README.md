
# GY-GY CLUB MONOLITH PROTOCOL [v.5.2.5]

## 🚨 CRITICAL CONSTRAINTS (MANDATORY)
* **Language**: English by default. A dropdown menu with 11 additional languages is mandatory. Never use other languages for the core interface unless explicitly requested.
* **Humor**: Humor and jokes are permanent attributes of buttons, language, and the logo.
* **Layout**: The GY-GY Logo and the "Back" button are mandatory permanent elements on every page.
* **Visuals**: 
    * Entrance Doors: max-height: 40vh.
    * The Horse: max-height: 35vh.
* **Marker "."**: The presence of the "." marker forbids editing established code blocks. Its absence allows for further modification.

## 🛠 ARCHITECTURAL STANDARDS
1. **Identity**: Unique digital trace (ID) hidden in the browser via Cookie-recognition.
2. **Registration**: Automatic assignment of a Numerical Code (Serial Number + random 4-digit number), a QR code, a unique Avatar or photo, and a Country Flag upon the first entry.
3. **Continuity**: A "Magic Key" (downloadable avatar or QR code) must be provided for cross-device access.
4. **Transition**: "Fading/Manifestation" effect with a mandatory 1.2s smooth transition between scenes.

## 🗺 NAVIGATION LOGISTICS
1. **Entrance**: Door visual -> Welcome Zone -> ID Generation -> Fade transition to the Hall.
2. **The Hall**: Slogan -> User Avatar with Flag -> "Proceed" dropdown menu -> Option "To the Bar".
3. **The Bar**: Slogan "The horse is a bartender too..." -> Bartender Horse visual -> Navigation to "Tables" or "AI Room".
4. **The Tables**: 6-table layout map + Microphone icon -> Audio 2.0 (Pouring sounds, horse coughing).
5. **The Server Room**: Dedicated audio file playback -> Transition to the AI Room.
